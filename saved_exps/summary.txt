# Scientific Analysis of SmartFallMM Fall Detection Experiments

## 1. Executive Summary

This analysis covers **12 distinct ablation studies** on wearable sensor-based fall detection using the SmartFallMM dataset (Texas State University). The experiments systematically evaluate: (1) sensor fusion methods, (2) model architectures, (3) loss functions, (4) preprocessing strategies, and (5) attention mechanisms.

**Key Finding**: The **Kalman Dual-Stream Balanced** achieves the best performance at **91.01% F1-score** (Young+Old population), followed by **Kalman Unified-Stream** at **90.32% F1**. For Young-only population, **Kalman Dual-Stream with Learned Gating** achieves **89.26% F1**.

**Document Organization**:
- **Section 2**: Architecture definitions with layer-by-layer descriptions
- **Section 3**: Dataset protocols (Young-only vs Young+Old)
- **Section 4**: Preprocessing pipeline and ablations
- **Section 5**: Experiments on Young-only population (30 subjects)
- **Section 6**: Experiments on Young+Old population (51 subjects)
- **Section 7**: Cross-population analysis and conclusions

## 2. Architecture Definitions

This section provides detailed layer-by-layer descriptions of all model architectures used in experiments. All models are implemented in PyTorch.

### 2.1 Unified-Stream Transformer (Code: `IMUTransformer`)

**Purpose**: Baseline transformer that processes all IMU channels jointly in a single stream.

**Input**: 6 channels [ax, ay, az, gx, gy, gz] where ax/ay/az are tri-axial accelerometer and gx/gy/gz are tri-axial gyroscope measurements.

**Architecture (Layer-by-Layer)**:
```
Layer 1: Input Projection
    Conv1d(in_channels=6, out_channels=64, kernel_size=1)
    Purpose: Project 6 sensor channels to 64-dimensional embedding space

Layer 2: Positional Encoding (Optional)
    Learnable positional embeddings added to sequence
    Purpose: Encode temporal position (often disabled for fall detection)

Layer 3-4: Transformer Encoder (2 layers)
    TransformerEncoderLayer(d_model=64, nhead=4, dim_feedforward=128)
    Each layer contains:
        - Multi-Head Self-Attention (4 heads, 16 dims each)
        - Feed-Forward Network: Linear(64→128) → GELU → Linear(128→64)
        - Residual connections + LayerNorm
    Purpose: Learn temporal dependencies across the 128-frame window

Layer 5: Temporal Pooling
    Global Mean Pooling across time dimension
    Input: (batch, 128 frames, 64 dims) → Output: (batch, 64 dims)
    Purpose: Aggregate temporal features into fixed-length vector

Layer 6: Classifier
    Linear(64, 1) → Sigmoid
    Purpose: Binary fall/non-fall prediction
```

**Parameters**: ~25K | **Typical F1**: 78-82%

---

### 2.2 Asymmetric Early-Fusion Transformer with Squeeze-Excitation (Code: `IMUTransformerSE`)

**Purpose**: Improved single-stream architecture with asymmetric capacity allocation and attention mechanisms.

**Input**: 6 channels [ax, ay, az, gx, gy, gz] split into acc (3ch) and gyro (3ch) streams before early fusion.

**Architecture (Layer-by-Layer)**:
```
Layer 1a: Accelerometer Projection (75% capacity)
    Conv1d(in_channels=3, out_channels=48, kernel_size=8, padding='same')
    BatchNorm1d(48)
    Purpose: Extract 48-dim features from accelerometer (higher capacity due to reliability)

Layer 1b: Gyroscope Projection (25% capacity)
    Conv1d(in_channels=3, out_channels=16, kernel_size=8, padding='same')
    BatchNorm1d(16)
    Dropout(0.4)  # Higher dropout for noisy gyro
    Purpose: Extract 16-dim features from gyroscope (lower capacity due to noise)

Layer 2: Early Fusion
    Concatenation: [acc_features(48d), gyro_features(16d)] → 64d
    LayerNorm(64)
    Purpose: Combine modalities before transformer (allows cross-modality attention)

Layer 3-4: Transformer Encoder (2 layers)
    TransformerEncoderLayer(d_model=64, nhead=4, dim_feedforward=128)
    Purpose: Learn joint acc+gyro temporal patterns

Layer 5: Squeeze-and-Excitation (SE) Block
    GlobalAvgPool: (batch, 128, 64) → (batch, 64)
    FC1: Linear(64, 16) → ReLU  (squeeze)
    FC2: Linear(16, 64) → Sigmoid  (excitation)
    Channel-wise multiplication: features × attention_weights
    Purpose: Learn adaptive channel importance (which channels matter for each sample)

Layer 6: Temporal Attention Pooling (TAP)
    Attention Network: Linear(64→32) → Tanh → Linear(32→1)
    Softmax over time dimension → weighted sum
    Purpose: Focus on fall-relevant time steps (impact phase ~500ms)

Layer 7: Classifier
    Linear(64, 1) → Sigmoid
```

**Parameters**: ~35K | **Typical F1**: 84-87%

**Key Innovations**:
- Asymmetric capacity (75/25) prioritizes reliable accelerometer
- SE attention learns which channels matter per sample
- TAP focuses on transient fall events within the window

---

### 2.3 Accelerometer-Only Transformer with Squeeze-Excitation (Code: `TransModelSE`)

**Purpose**: Single-modality baseline using only accelerometer data with SE attention.

**Input**: 4 channels [smv, ax, ay, az] where SMV = Signal Magnitude Vector = sqrt(ax² + ay² + az²)

**Architecture (Layer-by-Layer)**:
```
Layer 1: Input Projection
    Conv1d(in_channels=4, out_channels=64, kernel_size=8, padding='same')
    BatchNorm1d(64)
    Purpose: Project 4 accelerometer channels to 64-dim embedding

Layer 2-3: Transformer Encoder (2 layers)
    TransformerEncoderLayer(d_model=64, nhead=4, dim_feedforward=128)

Layer 4: Squeeze-and-Excitation Block
    Same structure as Section 2.2

Layer 5: Temporal Attention Pooling
    Same structure as Section 2.2

Layer 6: Classifier
    Linear(64, 1) → Sigmoid
```

**Parameters**: ~30K | **Typical F1**: 85-87%

---

### 2.4 Dual-Stream Asymmetric Transformer (Code: `DualStreamRobust`)

**Purpose**: Late-fusion architecture with completely separate processing paths for accelerometer and gyroscope, using asymmetric architectures per modality.

**Input**: 6 channels [ax, ay, az, gx, gy, gz] processed in parallel streams

**Architecture (Layer-by-Layer)**:
```
=== ACCELEROMETER STREAM (Strong Path) ===

Layer 1a: Depthwise Separable Convolution
    Depthwise Conv1d(3, 3, kernel=8, groups=3)  # Per-channel spatial filtering
    Pointwise Conv1d(3, 48, kernel=1)            # Channel mixing
    BatchNorm1d(48) → GELU → Dropout(0.3)
    Purpose: Efficient feature extraction with 8x fewer params than standard conv

Layer 2a: Transformer Encoder (1 layer)
    TransformerEncoderLayer(d_model=48, nhead=4, dim_feedforward=96)
    Purpose: Learn temporal patterns in accelerometer

Layer 3a: SE Block
    Squeeze: GlobalAvgPool → Linear(48, 12) → ReLU
    Excitation: Linear(12, 48) → Sigmoid
    Purpose: Channel attention for accelerometer features

Layer 4a: Temporal Attention Pooling
    Output: (batch, 48)


=== GYROSCOPE STREAM (Weak Path) ===

Layer 1b: Depthwise Separable Convolution
    Same structure but out_channels=16 (reduced capacity for noisy signal)

Layer 2b: SE Block (no transformer - would overfit on noise)
    Squeeze: Linear(16, 4) → ReLU
    Excitation: Linear(4, 16) → Sigmoid

Layer 3b: Simple MLP
    Linear(16, 16) → GELU → Dropout(0.5)

Layer 4b: Mean Pooling
    GlobalMeanPool over time → (batch, 16)


=== FUSION ===

Layer 5: Cross-Modal Gate
    Concatenate: [acc_feat(48), gyro_feat(16)] → 64
    Gate Network: Linear(64, 2) → Softmax
    Learned weighting: out = gate[0] * acc_proj + gate[1] * gyro_proj
    Purpose: Dynamic modality weighting based on input quality

Layer 6: Fusion MLP
    Linear(64, 48) → GELU → Dropout(0.3)

Layer 7: Classifier
    Linear(48, 1) → Sigmoid
```

**Parameters**: ~18K | **Typical F1**: 84-86%

**Key Innovations**:
- Asymmetric architectures: Full transformer for acc, simple MLP for gyro
- CrossModalGate learns to weight modalities dynamically
- Gyro stream intentionally limited to prevent noise overfitting

### 2.5 Kalman Dual-Stream with 65/35 Capacity (Code: `KalmanTransformerBaseline`)

**Purpose**: Dual-stream architecture for Kalman-fused 7-channel input with accelerometer-biased capacity.

**Input**: 7 channels [smv, ax, ay, az, roll, pitch, yaw] where roll/pitch/yaw are Kalman-estimated orientation.

**Architecture (Layer-by-Layer)**:
```
=== ACCELEROMETER STREAM (65% capacity) ===

Layer 1a: Accelerometer Projection
    Conv1d(in_channels=4, out_channels=41, kernel_size=8, padding='same')
    BatchNorm1d(41)
    Dropout(0.2)
    Purpose: Project [smv, ax, ay, az] to 41 dims (65% of 64)

Layer 2a: Transformer Encoder (2 layers)
    TransformerEncoderLayer(d_model=41, nhead=1, dim_feedforward=82)
    Note: Single head due to non-divisible dimension

Layer 3a: SE Block + TAP
    Same structure as previous, adapted to 41 dims


=== ORIENTATION STREAM (35% capacity) ===

Layer 1b: Orientation Projection
    Conv1d(in_channels=3, out_channels=23, kernel_size=8, padding='same')
    BatchNorm1d(23)
    Dropout(0.3)  # Higher dropout
    Purpose: Project [roll, pitch, yaw] to 23 dims (35% of 64)

Layer 2b: Transformer Encoder (2 layers)
    TransformerEncoderLayer(d_model=23, nhead=1, dim_feedforward=46)

Layer 3b: SE Block + TAP


=== FUSION ===

Layer 4: Concatenation Fusion
    Concat: [acc_feat(41), ori_feat(23)] → 64 dims
    LayerNorm(64)

Layer 5: Classifier
    Linear(64, 1) → Sigmoid
```

**Parameters**: ~50K | **Typical F1**: 88-89%

---

### 2.6 Kalman Dual-Stream with Learned Gating (Code: `KalmanGatedFusion`)

**Purpose**: Enhanced version with learned dynamic weighting between accelerometer and orientation streams.

**Input**: Same 7 channels as Section 2.5

**Architecture**: Same as Section 2.5 except:
```
=== MODIFIED FUSION ===

Layer 4: Gated Fusion (replaces concatenation)
    Acc Projection: Linear(41, 32) → ReLU
    Ori Projection: Linear(23, 32) → ReLU

    Gate Network:
        Concat: [acc_proj, ori_proj] → 64 dims
        Linear(64, 32) → ReLU → Dropout(0.3)
        Linear(32, 2) → Softmax  # [acc_gate, ori_gate]

    Weighted Fusion:
        out = acc_gate * acc_proj + ori_gate * ori_proj

    Purpose: Learn when to trust accelerometer vs orientation
             (e.g., accelerometer during impact, orientation during rotation)

Layer 5: LayerNorm(32)

Layer 6: Classifier
    Linear(32, 1) → Sigmoid
```

**Parameters**: ~52K | **Typical F1**: 89-90% (Young-only best)

---

### 2.7 Kalman Dual-Stream Balanced (Code: `KalmanBalancedRatio`)

**Purpose**: Equal capacity allocation between accelerometer and orientation streams.

**Input**: Same 7 channels

**Architecture**: Same as Section 2.5 except:
```
=== MODIFIED CAPACITY ===

Accelerometer Stream: 32 dims (50% instead of 65%)
Orientation Stream: 32 dims (50% instead of 35%)

All other layers scaled proportionally.
```

**Parameters**: ~50K | **Typical F1**: 91% (Young+Old best)

**Key Finding**: 50/50 capacity generalizes better to elderly population than 65/35.

---

### 2.8 Kalman Unified-Stream (Code: `KalmanSingleStream`)

**Purpose**: Baseline single-stream architecture for Kalman-fused input (no stream separation).

**Input**: 7 channels [smv, ax, ay, az, roll, pitch, yaw] processed jointly

**Architecture (Layer-by-Layer)**:
```
Layer 1: Input Projection
    Conv1d(in_channels=7, out_channels=64, kernel_size=8, padding='same')
    BatchNorm1d(64)
    Purpose: Project all 7 Kalman channels to 64-dim embedding

Layer 2-3: Transformer Encoder (2 layers)
    TransformerEncoderLayer(d_model=64, nhead=4, dim_feedforward=128)

Layer 4: SE Block

Layer 5: Temporal Attention Pooling

Layer 6: Classifier
    Linear(64, 1) → Sigmoid
```

**Parameters**: ~45K | **Typical F1**: 87-90%

### 2.9 Component Definitions

| Component | Full Name | Purpose | Implementation |
|-----------|-----------|---------|----------------|
| **SE** | Squeeze-and-Excitation | Channel attention | GlobalPool → FC → Sigmoid → Rescale |
| **TAP** | Temporal Attention Pooling | Focus on fall events | Attention scores → Softmax → Weighted sum |
| **DSConv** | Depthwise Separable Conv | Efficient convolution | Depthwise (per-channel) + Pointwise (1x1) |
| **CrossModalGate** | Cross-Modal Gating | Dynamic modality weighting | Softmax gate over concatenated features |

## 3. Dataset & Evaluation Protocol

### 3.1 Protocol A: Young-Only Population

| Specification | Value |
|---------------|-------|
| Dataset | SmartFallMM (Texas State University) |
| Subjects | 30 young adults (ages 18-35) |
| Sensor | Smartwatch (wrist-worn) |
| Modalities | Accelerometer (3-axis) + Gyroscope (3-axis) |
| Sampling Rate | ~30 Hz (variable, 25-35 Hz actual) |
| Task | Binary fall detection |
| Falls | 5 types (back, front, left, right, rotate) |
| ADLs | 9 activities (drinking, walking, waving, etc.) |
| Cross-Validation | 22-fold LOSO (Leave-One-Subject-Out) |
| Validation Subjects | [48, 57] (held out from training) |
| ADL Stride | 32 frames |
| Fall Stride | 16 frames |
| Window Size | 128 frames (~4 seconds) |
| Random Seed | 2 |

### 3.2 Protocol B: Young + Old Population

| Specification | Value |
|---------------|-------|
| Dataset | SmartFallMM (Texas State University) |
| Subjects | 30 young + 21 older adults (51 total) |
| Age Groups | Young (18-35) + Elderly (65+) |
| ADL Stride | 64 frames (slower elderly movements) |
| Elderly ADLs | TUG test, stepping up, etc. |
| Other params | Same as Protocol A |

**Key Difference**: Including elderly ADLs creates a more challenging, realistic test scenario. Elderly movements are slower and more variable, which can be confused with pre-fall patterns.

## 4. Preprocessing Pipeline & Ablations

### 4.1 Common Preprocessing Pipeline

```
Raw CSV → Timestamp Alignment → Gyro deg→rad → [Optional: Gravity Removal] →
Normalization → Kalman Fusion (if enabled) → Sliding Window → Class-Aware Stride
```

**Step Details**:

1. **Timestamp Alignment**: Resample accelerometer and gyroscope to common 30 Hz
2. **Gyroscope Conversion**: Convert degrees/second to radians/second
3. **Gravity Removal** (optional): High-pass Butterworth filter (0.3 Hz cutoff)
4. **Normalization**: Per-sample z-score: `x = (x - mean) / std` per channel
5. **Kalman Fusion** (if enabled): Fuse acc+gyro into orientation estimates [roll, pitch, yaw]
6. **Sliding Window**: Extract 128-frame windows
7. **Class-Aware Stride**: Fall stride=16, ADL stride=32/64

### 4.2 Gravity Removal Ablation

**Source**: `results/gravity_ablation_20251201_130609/`

**Method**: High-pass 4th-order Butterworth filter with 0.3 Hz cutoff removes DC gravity component from accelerometer, converting raw acceleration (ax, ay, az) to linear acceleration.

**Research Question**: Does removing gravity improve fall detection?

| Model | With Gravity (Raw) | Without Gravity | Δ F1 | Effect |
|-------|-------------------|-----------------|------|--------|
| Unified-Stream Transformer | 78.15 ± 17.45% | 80.93 ± 12.57% | **+2.78%** | **Improves** |
| Accelerometer-Only Transformer with SE | 83.30 ± 14.48% | 82.80 ± 12.04% | -0.50% | Negligible |

**Key Finding**: Gravity removal improves baseline models (+2.8% F1) but has negligible effect on SE-equipped models (-0.5% F1).

**Scientific Explanation**: Squeeze-and-Excitation attention learns to adaptively weight channels, effectively learning to ignore the gravity component when not useful. Models without SE benefit from explicit gravity removal as it reduces a confounding constant signal.

---

### 4.3 Normalization Ablation

**Sources**: `results/gyro_norm_comparison_20251201_154827/`, `results/se_norm_comparison_20251201_181534/`

**Method**: Per-sample z-score normalization applied to each channel independently.

#### 4.3.1 Overall Normalization Effect

| Normalization Strategy | F1 (%) | Δ vs None |
|------------------------|--------|-----------|
| All channels (acc + gyro) | 81.65 ± 9.87 | **+8.34%** |
| Gyroscope only | 79.69 ± 12.38 | +6.38% |
| None | 73.31 ± 16.36 | baseline |

**Key Finding**: Normalization provides substantial improvement (+8.34% F1), with gyroscope normalization being particularly important.

#### 4.3.2 Architecture-Specific Normalization Effect

| Architecture | With Normalization | Without Normalization | Better Option |
|--------------|-------------------|----------------------|---------------|
| Asymmetric Early-Fusion Transformer with SE | **84.9 ± 12.1%** | 80.3 ± 15.9% | **With** (+4.6%) |
| Dual-Stream Asymmetric Transformer | 79.0 ± 19.1% | **84.8 ± 9.6%** | **Without** (+5.8%) |

**CRITICAL FINDING**: Normalization effect is **architecture-dependent**:

- **Early-fusion models** (Asymmetric Early-Fusion Transformer with SE): Normalization helps by standardizing inputs before joint processing
- **Late-fusion dual-stream models** (Dual-Stream Asymmetric Transformer): Normalization **hurts** by removing magnitude information that the CrossModalGate uses for modality weighting

**Recommendation**: Enable normalization for single-stream/early-fusion architectures. Disable for late-fusion dual-stream with learned gating.

## 5. Experiments: Young-Only Population (30 subjects)

All experiments in this section use Protocol A (Young-only, ADL stride=32).

### 5.1 Kalman In-Model Fusion

**Research Question**: Does Kalman filter sensor fusion improve fall detection?

| Configuration | Filter Type | Parameters | Test Acc (%) | Test F1 (%) |
|--------------|-------------|------------|--------------|-------------|
| **Kalman Dual-Stream (Linear, Tuned)** | Linear | Tuned | 86.77 ± 8.13 | **88.09 ± 8.38** |
| Kalman Dual-Stream (Linear, Default) | Linear | Default | 86.50 ± 8.74 | 87.82 ± 8.94 |
| Kalman Dual-Stream (EKF, Tuned) | EKF | Tuned | 86.15 ± 7.79 | 87.50 ± 7.71 |
| Kalman Dual-Stream (EKF, Default) | EKF | Default | 84.88 ± 7.92 | 86.42 ± 7.90 |

**Statistical Tests**:
- Linear vs EKF (tuned): p=0.42, Cohen's d=-0.18 (not significant)
- Default vs Tuned (Linear): p=0.43, Cohen's d=0.17 (not significant)

**Key Finding**: Linear Kalman filter slightly outperforms Extended Kalman Filter (EKF), likely because fall detection doesn't require nonlinear state estimation. Parameter tuning provides marginal improvement (+0.27% F1).

---

### 5.2 Kalman Preprocessing Ablation

**Research Question**: Should we preprocess data with Kalman filtering before the model, or fuse within the model?

| Model | Description | Kalman Preprocessing | SE+TAP | Test F1 (%) |
|-------|-------------|---------------------|--------|-------------|
| Unified-Stream (Raw) | Single-stream | No | No | **84.40 ± 9.39** |
| Dual-Stream Asymmetric (Raw) | Dual-stream | No | No | **84.84 ± 7.40** |
| Unified-Stream + Kalman Preproc | Single + preproc | Yes | No | 77.55 ± 9.74 |
| Dual-Stream Asymmetric + Kalman Preproc | Dual + preproc | Yes | No | 79.47 ± 10.15 |
| Unified-Stream + Kalman + SE | Single + preproc | Yes | Yes | 79.88 ± 8.78 |
| Dual-Stream Asymmetric + Kalman + SE | Dual + preproc | Yes | Yes | 78.11 ± 11.69 |

**Key Finding**: **Kalman preprocessing HURTS performance** by ~6-7% F1.

**Scientific Explanation**: Kalman filtering acts as a low-pass filter that removes high-frequency information. Falls are characterized by **rapid accelerations and abrupt angular changes**—exactly the high-frequency components the filter removes. In-model fusion (Section 5.1) preserves raw accelerometer while adding orientation estimates, achieving the best of both worlds.

### 5.3 Loss Function Comparison (BCE vs Focal)

**Research Question**: Does Focal Loss improve performance on imbalanced fall data?

| Model | Loss Function | Test Acc (%) | Test F1 (%) |
|-------|---------------|--------------|-------------|
| Asymmetric Early-Fusion Transformer with SE | BCE | 82.54 ± 9.91 | 84.75 ± 10.03 |
| Asymmetric Early-Fusion Transformer with SE | **Focal** | 83.55 ± 9.07 | **86.57 ± 8.24** |
| **Dual-Stream Asymmetric Transformer** | **BCE** | **84.56 ± 9.35** | **85.81 ± 9.99** |
| Dual-Stream Asymmetric Transformer | Focal | 79.98 ± 9.41 | 83.72 ± 8.60 |

**Statistical Tests**:
- SE+Focal vs SE+BCE: p=0.20, Δ=+1.82% F1 (not significant, small effect)
- DualStream+BCE vs DualStream+Focal: p=0.005**, Δ=+4.58% Acc (significant, medium effect d=0.67)

**Key Finding**: Loss function choice is **architecture-dependent**:
- **Single-stream (Asymmetric Early-Fusion with SE)**: Focal Loss helps (+1.82% F1) by focusing on hard examples
- **Dual-stream (Dual-Stream Asymmetric Transformer)**: BCE works significantly better; Focal destabilizes training in complex architectures

### 5.4 Positional Encoding Ablation

**Research Question**: Does positional encoding help transformers for fall detection?

| Model | Positional Encoding | Test Acc (%) | Test F1 (%) |
|-------|---------------------|--------------|-------------|
| Accelerometer-Only Transformer with SE (baseline) | Yes | **85.88 ± 8.49** | **87.82 ± 8.24** |
| Unified-Stream (No Pos) | No | 83.08 ± 9.80 | 85.88 ± 8.38 |
| Unified-Stream (With Pos) | Yes | 79.50 ± 12.20 | 83.56 ± 9.96 |
| Dual-Stream (No Pos) | No | 81.76 ± 10.12 | 84.35 ± 9.61 |
| Dual-Stream (With Pos) | Yes | 78.24 ± 15.77 | 81.44 ± 13.90 |

**Key Finding**: **Positional encoding HURTS performance** by 2-3% F1.

**Scientific Explanation**: Fall detection relies on **motion patterns** (acceleration spikes, rotational changes), not **absolute temporal position** within a window. Falls can occur at any point in the window—their characteristic signature is the same regardless of position. Adding positional encoding introduces unnecessary inductive bias that causes overfitting.

**Note**: The Accelerometer-Only Transformer with SE baseline outperforms variants due to its SE attention mechanism, not positional encoding.

### 5.5 Squeeze-Excitation Module & SMV Ablation

**Research Question**: How important are Squeeze-and-Excitation (SE) blocks and Signal Magnitude Vector (SMV)?

| Model | SE Block | SMV Input | Test F1 (%) |
|-------|----------|-----------|-------------|
| Accelerometer-Only Transformer with SE | Yes | No | **86.37%** |
| Accelerometer-Only Transformer with SE + SMV | Yes | Yes | 85.54% |
| Unified-Stream Transformer (baseline) | No | No | 76.29% |
| Unified-Stream Transformer + SMV | No | Yes | 81.28% |

**Key Findings**:
1. **SE blocks provide ~10% F1 improvement** (76.29% → 86.37%)
2. **SMV alone provides +4.99% improvement** without SE
3. **SMV is redundant when SE is present** (-0.83% when added to SE model)

**Scientific Explanation**: SE attention learns to dynamically weight channels based on their informativeness for each sample. SMV provides a static, orientation-invariant magnitude feature. When SE is present, it learns an adaptive version of what SMV provides statically—making explicit SMV computation redundant and slightly harmful (adds noise).

### 5.6 Modality Comparison

**Research Question**: Which input modalities work best?

| Configuration | Channels | Description | Test F1 (%) |
|--------------|----------|-------------|-------------|
| Accelerometer only | 3ch [ax, ay, az] | Tri-axial accelerometer | ~85% |
| Accelerometer + SMV | 4ch [smv, ax, ay, az] | + magnitude | ~82% |
| Accelerometer + Gyroscope | 6ch [ax, ay, az, gx, gy, gz] | Full IMU | ~82% |
| Full (all) | 7ch [smv, ax, ay, az, gx, gy, gz] | + SMV | ~78% |

**Key Finding**: **Simpler may be better**. Accelerometer-only achieves comparable or better performance than full 7-channel input.

**Scientific Explanation**: Falls generate distinctive accelerometer signatures (impact spikes, free-fall phases). Gyroscope data is noisier and may introduce confounding information. Adding more channels increases model complexity and potential for overfitting on the limited dataset (~22 training subjects per fold).

### 5.7 Single vs Dual-Stream Architecture (Young-Only)

**Research Question**: Is separate processing of accelerometer/gyroscope (dual-stream) better than joint processing (single-stream)?

| Experiment | Best Single-Stream F1 | Best Dual-Stream F1 | Winner |
|------------|----------------------|---------------------|--------|
| Stream Ablation (Raw) | 84.40% | 84.84% | Dual (+0.44%) |
| Positional Encoding | **87.82%** | 84.35% | **Single (+3.47%)** |
| BCE/Focal Loss | **86.57%** | 85.81% | **Single (+0.76%)** |
| SE/SMV Comparison | **86.37%** | 61.61%* | **Single** |

*Dual-stream training failed in this experiment

**Conclusion**: **Single-stream architectures win 3/4 experiments** for Young-only population with raw acc+gyro input.

**Reasons**:
1. Joint learning allows transformer attention to discover cross-modality patterns
2. SE attention provides implicit modality weighting without stream separation
3. Fewer parameters = less overfitting with limited training data
4. Dual-stream requires careful capacity balancing between streams

### 7.4 What Works

1. **In-model Kalman fusion** (preserving raw accelerometer) outperforms preprocessing by ~4-6% F1
2. **Squeeze-and-Excitation attention** provides ~5-10% improvement for young-only (architecture-dependent)
3. **Class-aware stride** (16 for falls, 32/64 for ADLs) addresses class imbalance effectively
4. **Per-sample z-score normalization** for single-stream/early-fusion architectures
5. **Balanced capacity allocation (50/50)** generalizes better to elderly population
6. **Temporal Attention Pooling** focuses on transient fall events

### 7.5 What Doesn't Work

1. **Kalman preprocessing** removes discriminative high-frequency fall signatures (-6-7% F1)
2. **Positional encoding** adds unhelpful bias—falls are position-invariant (-2-3% F1)
3. **SMV with SE** is redundant—SE learns adaptive magnitude weighting
4. **Asymmetric capacity (75/25)** hurts performance significantly (-5% F1)
5. **Deep/complex architectures** (3+ layers, uncertainty features) hurt with limited data
6. **Normalization for late-fusion dual-stream** removes magnitude information used by gating

## 8. Methodological Notes

### Strengths
- **22-fold LOSO CV**: Robust subject-level generalization testing
- **Fixed validation set**: Consistent hyperparameter tuning across experiments
- **Statistical tests**: Paired t-tests with effect sizes (Cohen's d)
- **Multiple ablations**: Systematic component analysis (12 experiments)
- **Two populations**: Young-only and Young+Old for generalization testing

### Limitations
- High variance (±8-10% F1) across folds indicates subject-dependent performance
- Some dual-stream experiments had training failures
- Smartwatch sensor variability (25-35 Hz actual sampling rate)
- Limited to wrist-worn sensors; other placements may differ
